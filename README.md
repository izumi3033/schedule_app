# 研究スケジュールプランナー

研究の予定を「年間」「3ヶ月」「月間」の3つのビューで管理できる、Electron製のデスクトップアプリです。データはすべてローカルに保存されます。

## 主な機能

- 年間・3ヶ月・月間の3種類のスケジュールビュー切り替え（各ビューは表示位置が独立しており、互いに影響しません）
- 予定の新規作成・編集・削除（タイトル、詳細メモ、開始日/終了日、タグ、色、進捗）
- 予定ごとのスモールゴール（小さな目標）の追加・編集・完了チェック・削除。達成予定日を付けると月間ビューに独立した予定として表示されます
- タグの追加・編集（名称・色の変更）・削除（「タグ管理」から一括管理）、予定ごとに1つ選択
- 予定の検索（サイドバーのフィルタ）
- 進捗は年間・3ヶ月ビューのカレンダー上で視覚的に確認（塗りつぶし＝進捗、赤い斜線＝期限超過分）。進捗値自体は編集モーダルから手動で設定します
- 「今日」への移動や前後期間へのナビゲーション
- Ctrl+Z（Macは⌘+Z）での操作の取り消し、Shift+Ctrl+Z（⌘+Shift+Z）でのやり直し
- アプリのユーザーデータディレクトリへのローカルJSON保存（サーバー通信なし）

## 技術構成

- [Electron](https://www.electronjs.org/)（メインプロセス: `main.js` / プリロード: `preload.js`）
- フロントエンドはプレーンな HTML / CSS / JavaScript（`renderer/`）
- データ保存先: `app.getPath('userData')/schedule-data.json`

## Googleカレンダー連携（読み取り専用）

Googleカレンダーの予定を月間ビューに表示できます（アプリからGoogle側への書き込みは行いません）。

**初回セットアップ（約10分・一度だけ）**

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. 「APIとサービス → ライブラリ」で **Google Calendar API** を有効化
3. 「APIとサービス → OAuth同意画面」を設定（User Type: 外部、テストユーザーに自分のGoogleアカウントを追加）
4. 「APIとサービス → 認証情報 → 認証情報を作成 → OAuthクライアントID」で、アプリの種類に **「デスクトップアプリ」** を選んで作成
5. 表示されたクライアントIDとクライアントシークレットを、アプリのサイドバー「Googleカレンダー連携」から入力し、「Googleでログイン」

認証情報とトークンは `userData` ディレクトリにローカル保存されます。OAuth同意画面が「テスト」ステータスの場合、約7日ごとに再ログインが必要です（同意画面を「本番」に公開すると回避できます）。

## Web版で確認する（開発用）

Electronを起動せずブラウザ上でUIを確認・開発できます。

```bash
npm run web
```

`http://localhost:5500` をブラウザで開いてください。データは `schedule-data.json` ではなくブラウザの localStorage に保存され、Electron版とは共有されません（開発・見た目確認のみを想定した簡易サーバーです）。

## ディレクトリ構成

```
.
├── main.js          # Electronメインプロセス（ウィンドウ生成・データ読み書き）
├── preload.js        # レンダラーへ安全にAPIを公開（contextBridge）
├── serve-web.js       # ブラウザ確認用の簡易静的サーバー（npm run web）
├── package.json
└── renderer/
    ├── index.html     # UI構造
    ├── style.css      # スタイル
    ├── app.js         # UIロジック・ビュー描画・状態管理
    └── web-shim.js    # ブラウザ実行時にscheduleAPIをlocalStorageで代替
```
